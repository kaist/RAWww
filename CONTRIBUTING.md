# Как участвовать в разработке

Спасибо, что решили заглянуть внутрь Контрольки. Проект управляется через `uv`,
требует Python 3.12 или новее и использует PySide6.

Перед изменениями прочитайте [описание архитектуры](docs/ARCHITECTURE.md). Оно
поможет быстро понять границы между интерфейсом, кэшем, декодированием и
ShotSync.

## Подготовка окружения

```powershell
uv sync
uv run rawww
```

Приложение можно сразу открыть на тестовой папке или фотографии:

```powershell
uv run rawww "D:\photos\test-shoot"
uv run rawww "D:\photos\test-shoot\IMG_0001.CR3"
```

Не добавляйте в репозиторий `.venv`, кэши, локальные `.env`, папки сборки и
пользовательские фотографии.

## Перед отправкой изменения

```powershell
uv run python -m unittest discover -s tests -v
```

Проверьте также сценарий вручную, если изменение касается Qt, drag-and-drop,
нативных диалогов, мультимедиа или полноэкранного режима: unit-тест умеет многое,
но пока не научился недовольно смотреть на мигающее окно.

## Стиль изменений

- Делайте небольшие изменения с одной понятной целью.
- Новую бизнес-логику по возможности оформляйте чистой функцией и тестируйте
  отдельно от Qt.
- Не выполняйте долгую работу в UI-потоке.
- Комментарии и докстринги пишите по-русски. Объясняйте причину, ограничения и
  владение состоянием; очевидный код переводить на русский прозой не требуется.
- Умеренный юмор приветствуется, если он не скрывает технический смысл.
- Пользовательские команды и поведение отражайте в `README.md`.
