# AGENTS.md

## Проект

«Контролька» — настольное приложение на Python 3.12 и PySide6 для быстрого
просмотра, отбора и синхронизации фотографий. Окружением и зависимостями
управляет `uv`.

Перед крупным изменением прочитайте:

- `docs/ARCHITECTURE.md` — устройство приложения и поток данных;
- `CONTRIBUTING.md` — правила разработки и проверки;
- `README.md` — пользовательские возможности и команды.

## Команды

- Создать или обновить окружение: `uv sync`
- Запустить приложение: `uv run rawww`
- Открыть конкретный путь: `uv run rawww "D:\photos\shoot"`
- Запустить модуль: `uv run python -m <module>`
- Выполнить тесты: `uv run python -m unittest discover -s tests -v`
- Добавить зависимость: `uv add <package>`
- Добавить зависимость разработки: `uv add --dev <package>`
- Собрать переносимую Windows-версию:
  `uv run python scripts/build_pyinstaller.py --portable`

## Карта кода

- `src/rawww/app.py` — Qt-интерфейс и координация пользовательских сценариев.
- `MainWindow` владеет вкладками; каждой вкладке принадлежит свой `Workspace`.
- `Workspace` владеет текущей папкой, представлениями и жизненным циклом
  фоновых запросов, но не должен содержать алгоритмы декодирования или сети.
- `imaging.py`, `decode_scheduler.py`, `decode_cache.py` — декодирование,
  приоритеты и оперативный кэш.
- `cache.py` — одноразовый SQLite-кэш папок; он не является источником
  пользовательских файлов.
- `exif.py` и `xmp.py` — чтение метаданных и запись sidecar-файлов.
- `ai.py`, `face_analysis.py`, `face_sets_sync.py` — эмбеддинги, лица и наборы.
- `shotsync_*.py` — HTTP, WebSocket, загрузка, приём, локальный отбор и UI
  интеграции ShotSync.
- `dialogs.py`, `widgets.py`, `theme.py` — переиспользуемый интерфейс и стили.

## Qt, потоки и процессы

- Не блокируйте главный поток диском, сетью, ExifTool, AI или большим циклом.
- Виджеты, `QPixmap` и прочие UI-объекты изменяются только в главном потоке.
- Между процессами передавайте байты и простые структуры, а не виджеты Qt.
- Фоновый результат обязан проверить актуальную папку, поколение запроса и
  состояние закрытия перед изменением интерфейса.
- При завершении сначала помечайте владельца закрытым, затем останавливайте
  исполнителей: поздний callback не должен попасть в уже выключенную очередь.
- Виджет формулирует намерение сигналом; файловую или сетевую операцию выполняет
  координатор. Например, `PhotoGrid` не должен сам перемещать файлы.

## Кэш и файлы

- Считайте содержимое `FolderCache` производным и восстанавливаемым.
- Записывайте XMP, скачанные файлы и результаты экспорта атомарно через
  временный файл, если частичная запись может быть замечена приложением.
- Перед переименованием или переносом открытой папки закрывайте её SQLite-базу,
  особенно на Windows.
- Не удаляйте и не перемещайте пользовательские файлы без явного действия
  пользователя.
- Не добавляйте в Git виртуальные окружения, кэши, `.env`, пользовательские
  фотографии и результаты сборки.

## Комментарии и документация

- Комментарии и докстринги пишутся на русском языке.
- Для класса опишите ответственность, владение состоянием и границу с соседними
  компонентами.
- Для функции объясните результат, важный побочный эффект, ограничение или
  причину выделения в отдельный шаг.
- Не пересказывайте очевидную строку кода. Комментарий должен отвечать на
  «почему?» или «что здесь легко сломать?».
- Лёгкий юмор допустим, если технический смысл остаётся однозначным. В обработке
  ошибок, данных пользователя и безопасности сначала ясность, потом стендап.
- Новое пользовательское поведение отражайте в `README.md`; архитектурное
  решение — в `docs/ARCHITECTURE.md`.
- Все новые строки интерфейса оборачивайте в `_()` из `rawww.i18n` и переводите
  на все поддерживаемые языки: после правок выполните
  `uv run python scripts/update_translations.py all`, заполните появившиеся
  `msgstr` во всех `src/rawww/locale/<язык>/LC_MESSAGES/rawww.po` и снимите
  пометки `#, fuzzy` с ошибочно подставленных переводов. Не оставляйте новые
  строки без перевода.

## Проверки

- После изменения Python-кода выполните как минимум:
  `uv run python -m compileall -q src scripts tests`.
- Затем запустите релевантные тесты или весь набор `unittest`.
- Для Qt, drag-and-drop, видео, полноэкранного режима и нативных диалогов
  дополнительно проверьте сценарий вручную.
- После изменений фоновых очередей проверьте быструю смену папки, закрытие
  вкладки во время работы и активную прокрутку большой директории.
- Перед завершением выполните `git diff --check`.

## Изменения и Git

- Перед широкими правками проверьте `git status --short`.
- Не откатывайте несвязанные пользовательские изменения.
- Делайте небольшие, сфокусированные изменения с одной понятной целью.
- Не используйте разрушительные команды Git без прямого указания пользователя.
