# Контролька

Контролька — настольное приложение для быстрого просмотра, отбора и
подготовки фотографий. Оно открывает папки с RAW, JPEG и другими
поддерживаемыми изображениями, помогает расставлять оценки и метки, а затем
передаёт результат в привычный рабочий процесс через XMP. Мышь пригодится,
но горячие клавиши будут рады сделать большую часть работы за неё.

Подробное описание для фотографов: [shotsync.ru/ctrlka](https://shotsync.ru/ctrlka).

## Возможности

- Быстрый просмотр RAW и JPEG, полноэкранный режим, лента кадров и несколько
  вкладок с папками.
- Переключатель ориентации в панели зума сетки: одна кнопка переводит всю
  раскладку из альбомной в портретную и обратно. В вертикальном режиме карточки
  сетки и ленты превью вытягиваются под вертикальные снимки, а нижняя лента
  полноэкранного просмотра переезжает вертикально вправо. Выбор сохраняется
  между запусками и запоминается отдельно для каждой папки: при повторном
  заходе восстанавливается её ориентация, а новые папки открываются в последней
  выбранной.
- Контекстное меню папок в дереве и сетке позволяет открыть папку в отдельной
  вкладке; для файлов в сетке на Windows доступно системное меню Проводника.
- Активная папка получает приоритет фоновой обработки: скрытые вкладки не
  отнимают очередь превью и AI, а недостающие результаты продолжаются из кэша
  после возвращения.
- Отбор оценками, цветными метками, комментариями, тегами и быстрыми метками;
  копирование, перемещение и удаление файлов прямо из приложения.
- Быстрое копирование и перемещение по `Shift+C` / `Shift+M`: после выбора цели
  задача уходит в общую фоновую очередь. Боковая панель показывает файлы,
  объём, скорость и ETA, позволяет поставить работу на паузу или отменить её.
  Совпавшие имена по умолчанию автоматически получают свободный номер; оба
  поведения меняются в настройках.
- Импорт с карты памяти через кнопку над съёмными томами или `Ctrl+Shift+I`:
  можно выбрать одну или несколько карт, папку, дату или название съёмки;
  структуру, удаление исходников и резервную копию задают в «Настройки → Поведение».
  Несколько карт копируются одновременно в доступных слотах очереди.
- Ручной порядок кадров: перетащите карточку в сетке, после чего в сортировке
  появится пункт «Пользовательский»; порядок запоминается отдельно для папки.
- Групповые операции: переименование, изменение размера изображений и
  пересохранение JPEG.
- Двусторонняя синхронизация XMP: рейтинги, цветовые метки, комментарии,
  ключевые слова и найденные лица читаются и записываются без удаления настроек
  проявки и других незнакомых полей. Изменения Lightroom или Capture One
  подхватываются автоматически, а одновременные правки разрешаются по полям.
  RAW+JPEG с одинаковым именем используют один общий sidecar.
- Локальный AI-анализ: автоматический многопрототипный поиск профиля и анфаса по
  одному клику, наборы лиц и объединение похожих кадров в серии.
- Определение закрытых глаз: кнопка-переключатель «Закрытые глаза» рядом с
  фильтром крупности плана помогает отсеять брак. Состояние глаз считается по
  разметке лица (eye aspect ratio каждого глаза), поэтому подмигивание не
  бракуется — закрытым лицо считается только когда закрыты оба глаза. Кадр
  бракуется, если закрыты глаза у крупнейшего лица, либо когда в кадре не больше
  трёх лиц и глаза закрыты хотя бы у одного из них. В больших группах учитывается
  только крупнейшее лицо, чтобы моргнувший на фоне не браковал кадр.
- AI-фильтры («Закрытые глаза», крупность плана) действуют
  только там, где есть данные анализа. При переходе в папку без AI-анализа
  включённый ранее AI-фильтр сбрасывается, чтобы список не оказался пустым.
- Управление цветом в полном просмотре: кадр переводится в ICC-профиль монитора,
  чтобы цвета были верными на калиброванном или широкогамутном дисплее. Профиль
  берётся из системы автоматически (или задаётся вручную), поддерживаются
  rendering intent и компенсация чёрной точки. Настраивается в
  «Настройки → Интерфейс → Управление цветом»; коррекция работает и в зуме.
- Аудиозаметки, записанные камерой, рядом со снимком.
- Локальный журнал необработанных ошибок приложения и фоновых потоков: «Настройки → О приложении → Лог ошибок» позволяет просмотреть, скопировать или очистить его. В переносимой версии он хранится в `work/logs`, в установленной — в пользовательском каталоге данных ОС.
- Интеграция с ShotSync: отправка съёмки на отбор, получение фотографий в
  реальном времени, синхронизация оценок и меток между устройствами.
- Мультиязычный интерфейс: русский, английский, немецкий и упрощённый
  китайский. По умолчанию берётся язык системы, а если он не поддерживается —
  английский. Язык можно задать вручную в «Настройки → Интерфейс»; новый выбор
  применяется после перезапуска.

## Требования

- Python 3.12 или новее;
- [uv](https://docs.astral.sh/uv/);
- Windows, macOS или Linux с графической средой. Основной сценарий сборки и
  интеграции с проводником реализован для Windows.

Модели распознавания лиц и ExifTool поставляются вместе с приложением. В
собранных macOS- и Linux-версиях ExifTool содержит свой Perl runtime, поэтому
устанавливать ExifTool или Perl отдельно не нужно. При запуске из исходников на
macOS/Linux нужен Perl только для разработки. Первое использование AI и метаданных может занять немного времени: приложение
загружает соответствующие компоненты только тогда, когда они понадобились.

## Установка и запуск из исходников

Перейдите в корень репозитория и попросите `uv` собрать окружение. Отдельно
создавать `.venv` и вручную ставить зависимости не нужно — пусть рутиной
занимается инструмент, ему за это даже не приходится платить кофе.

```powershell
uv sync
uv run rawww
```

`uv sync` создаст локальное виртуальное окружение, установит зависимости из
`pyproject.toml` и зафиксированные версии из `uv.lock`. Для повторного запуска
достаточно команды `uv run rawww`.

Можно сразу открыть папку или отдельный кадр:

```powershell
uv run rawww "D:\фото\съёмка"
uv run rawww "D:\фото\съёмка\IMG_0001.CR3"
```

## Как пользоваться

1. Откройте папку с фотографиями или перетащите её в приложение.
2. Просматривайте кадры в сетке или на весь экран; оставляйте оценки, цветные
   метки и комментарии.
3. При необходимости включите AI-анализ, чтобы собрать серии и найти лица.
4. Включите автоматическую синхронизацию XMP для дальнейшей работы в редакторе
   либо синхронизируйте съёмку с ShotSync. В ручном режиме меню XMP содержит
   две операции: «Прочитать XMP» принимает данные sidecar, а «Записать XMP» —
   данные Контрольки. При включённой автоматике обе команды недоступны.

Сочетания клавиш настраиваются в самом приложении. Полный список и описание
интерфейса находятся в разделе помощи и на странице проекта. По умолчанию
`Ctrl+Shift+N` создаёт новую папку в текущей директории.

В окне быстрого копирования или перемещения выбранную папку назначения можно
подтвердить повторным сочетанием, клавишей `Enter` либо цифрой `1–9`. Пока есть
активная или ожидающая операция, её общая очередь видна ниже избранного;
после завершения последней задачи панель скрывается.

Импорт показывает только смонтированные съёмные тома. Для нескольких выбранных
карт дата каталога определяется по EXIF первого снимка первой карты. Служебные
записи камер отсекаются по расширению, а не по названию каталога. По умолчанию
все файлы складываются в один каталог; при отключении опции сохраняется
относительная структура карты без пустых папок. Резервная копия запускается
только после успешного основного импорта.

## ShotSync

ShotSync не обязателен: локальный отбор работает без учётной записи и без
интернета. При входе в ShotSync можно отправить папку на удалённый отбор,
получать новые кадры со съёмки и синхронизировать оценки, цветовые метки,
комментарии, коды замены и наборы лиц. Подробнее — на
[shotsync.ru/ctrlka](https://shotsync.ru/ctrlka).

## Разработка

Если вы впервые открыли исходники, двигайтесь в таком порядке:

1. Прочитайте [описание архитектуры](docs/ARCHITECTURE.md) — там есть карта
   классов, жизненный цикл папки, фоновые очереди, кэш и ShotSync.
2. Посмотрите [правила участия](CONTRIBUTING.md) — подготовка окружения, стиль
   изменений, ручные проверки и условия GPL.
3. Откройте `src/rawww/app.py` и найдите `MainWindow`, `Workspace`, `PhotoGrid`,
   `FullView` и `ViewerStrip`. Это основные точки сборки интерфейса.
4. Алгоритмы ищите в профильных модулях: `imaging.py`, `cache.py`,
   `decode_scheduler.py`, `exif.py`, `ai.py`, `xmp.py` и `shotsync_*.py`.

Быстрая подготовка рабочего окружения разработчика:

```powershell
uv sync
uv run python -m compileall -q src scripts tests
uv run python -m unittest discover -s tests -v
uv run rawww
```

Все проектные комментарии, докстринги и документы пишутся по-русски. Имена
API, классов, форматов и инструментов сохраняются в оригинальном виде, иначе
поиск по документации быстро превратится в игру «угадай перевод термина».

## Проверка изменений

```powershell
uv run python -m unittest discover -s tests -v
```

Для небольшой правки можно запустить только связанный тестовый модуль, но перед
публикацией изменения лучше выполнить весь набор. Изменения Qt-интерфейса,
drag-and-drop, видео и нативных окон дополнительно проверяются вручную.

Для профилирования доступны вспомогательные команды:

```powershell
uv run python -m rawww.ai_benchmark --limit 32
uv run python -m rawww.benchmark "D:\фото\съёмка" --limit 30 --full-limit 8 --full-size 2560
```

## Переводы

Строки интерфейса помечаются вызовом `_()` из `rawww.i18n` (исходный текст —
русский, он же служит ключом). Каталоги переводов лежат в
`src/rawww/locale/<язык>/LC_MESSAGES/rawww.po`, скомпилированные `.mo` собираются
рядом. Рабочий процесс через Babel:

```powershell
uv run python scripts/update_translations.py extract   # обновить rawww.pot
uv run python scripts/update_translations.py update     # влить новые строки в .po
uv run python scripts/update_translations.py compile    # собрать .mo
```

Команда `all` выполняет все три шага подряд. После правок `.po` не забудьте
`compile`: приложение и сборка используют именно `.mo`.

## Сборка для Windows

```powershell
uv run python scripts/build_pyinstaller.py --portable
```

Опция `--portable` создаёт переносимую сборку. Скрипт сборки сам подготавливает
необходимые ресурсы и версию приложения. Для регистрации собранной программы
в проводнике Windows используйте:

```powershell
uv run python scripts/register_windows_integration.py
```

Чтобы отменить регистрацию:

```powershell
uv run python scripts/register_windows_integration.py --unregister
```

В собранной версии в «Настройки → Поведение → Интеграция с Проводником» можно
зарегистрировать Контрольку для JPG и RAW как кандидат на приложение по
умолчанию. Windows откроет системную страницу, где пользователь подтверждает
выбор расширений сам.
